LM

LLMs & Chat Models

Core language model wrappers
Khái niệm Wrapper chuẩn cho provider model; output thường là AIMessage.
Khi dùng Gọi model trực tiếp, chain, agent, classification, extraction.
Constructor init_chat_model(model, temperature, max_tokens, timeout)
Methods invoke, stream, batch, with_structured_output
from langchain.chat_models import init_chat_model

model = init_chat_model(
    "openai:gpt-5.5",
    temperature=0.2,
    max_tokens=800,
    timeout=30,
    max_retries=6,
)

msg = model.invoke([
    {"role": "system", "content": "Answer in Vietnamese."},
    {"role": "user", "content": "LangChain la gi?"},
])

for token in model.stream("Explain LCEL"):
    print(token.content, end="")

Best: đặt key qua env, cấu hình timeout/retry, log usage và latency.

Mistake: hard-code API key hoặc giả định mọi provider hỗ trợ tool calling như nhau.

PT

Prompt Templates

Dynamic prompt construction
Khái niệm Template cho system/human/AI messages có biến.
Khi dùng Reuse instruction, chèn history, tạo chain có input rõ.
Constructor from_messages, from_template
Methods invoke, format_messages, partial, append
from langchain_core.prompts import (
    ChatPromptTemplate,
    MessagesPlaceholder,
)

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a {role}."),
    MessagesPlaceholder("history"),
    ("human", "{question}"),
])

messages = prompt.invoke({
    "role": "LangChain tutor",
    "history": [],
    "question": "What is Runnable?",
})

Best: đặt tên biến rõ, system prompt cụ thể, tách prompt khỏi business logic.

Mistake: dùng f-string với input user chưa kiểm soát hoặc quên truyền biến.

OP

Output Parsers

Parse and structure LLM output
Khái niệm Chuyển output model sang string, JSON hoặc object app cần.
Khi dùng LCEL nhỏ, model chưa có native structured output.
Constructor StrOutputParser(), JsonOutputParser()
Methods parse, invoke, get_format_instructions
from langchain_core.output_parsers import (
    StrOutputParser,
    JsonOutputParser,
)

parser = JsonOutputParser()
prompt = ChatPromptTemplate.from_template(
    "Return JSON only: {topic}\n{format}"
)

chain = prompt | model | parser
data = chain.invoke({
    "topic": "retriever",
    "format": parser.get_format_instructions(),
})

Best: với output quan trọng, ưu tiên schema Pydantic + structured output.

Mistake: regex JSON từ text tự nhiên hoặc không bắt lỗi parse.

CH

Chains - LCEL

Composable pipeline with pipe operator
Khái niệm Runnable là interface chung cho prompt, model, parser, retriever.
Khi dùng Flow cố định: prompt -> model -> parser; map/merge nhiều nhánh.
Constructor RunnableLambda(fn), RunnableParallel(...)
Methods invoke, ainvoke, batch, stream, with_retry
from langchain_core.runnables import RunnableLambda, RunnableParallel
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_template(
    "Summarize {topic} in 3 bullets, Vietnamese."
)

chain = prompt | model | StrOutputParser()
print(chain.invoke({"topic": "LangChain Runnable"}))

parallel = RunnableParallel(
    summary=chain,
    upper=RunnableLambda(lambda x: x["topic"].upper()),
)

print(parallel.invoke({"topic": "LCEL"}))

reliable = chain.with_retry(stop_after_attempt=3)

Best: dùng chain cho tác vụ deterministic, dễ test và dễ stream.

Mistake: dùng agent cho một pipeline đơn giản hoặc không kiểm soát kiểu input/output.

ME

Memory

Conversation history management
Khái niệm Short-term memory lưu state theo conversation thread.
Khi dùng Chat nhiều lượt, resume agent, user context.
Constructor create_agent(..., checkpointer=...)
Parameter thread_id trong configurable
from langchain.agents import create_agent
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    model="openai:gpt-5.5",
    tools=[],
    checkpointer=InMemorySaver(),
)

config = {"configurable": {"thread_id": "user-42"}}

agent.invoke({"messages": [{"role": "user",
    "content": "My name is Diem."}]}, config)

result = agent.invoke({"messages": [{"role": "user",
    "content": "What is my name?"}]}, config)

Best: production dùng Postgres/SQLite checkpointer và trim/summarize history.

Mistake: dùng InMemorySaver cho production hoặc đổi thread_id mỗi request.

EM

Embeddings

Text to vector representations
Khái niệm Biến text thành vector để tìm kiếm ngữ nghĩa.
Khi dùng RAG, semantic search, clustering, duplicate detection.
Constructor OpenAIEmbeddings(model=...)
Methods embed_query, embed_documents
from langchain_openai import OpenAIEmbeddings

emb = OpenAIEmbeddings(
    model="text-embedding-3-small"
)

query_vec = emb.embed_query("What is RAG?")
doc_vecs = emb.embed_documents([
    "Retrieval augments generation.",
    "Tools let agents call functions.",
])

Best: dùng cùng embedding model cho index và query, lưu version model.

Mistake: trộn vectors từ nhiều model hoặc embed raw HTML/noise.

VS

Vector Stores

Store and search embeddings
Khái niệm Lưu vector + metadata, tìm documents tương tự.
Khi dùng Knowledge base, semantic search, RAG corpus.
Constructor InMemoryVectorStore(embedding=emb)
Methods add_documents, similarity_search, as_retriever
from langchain_core.vectorstores import InMemoryVectorStore

store = InMemoryVectorStore(emb)
store.add_documents(chunks)

docs = store.similarity_search(
    "How do agents use tools?",
    k=4,
)

retriever = store.as_retriever(
    search_kwargs={"k": 4}
)

Best: production chọn store có persistence, filtering, backup.

Mistake: dùng in-memory store cho dữ liệu thật hoặc thiếu source metadata.

RT

Retrievers

Fetch relevant docs for RAG
Khái niệm Nhận query, trả về danh sách Document liên quan.
Khi dùng 2-step RAG, agentic RAG, hybrid search.
Constructor vectorstore.as_retriever(...)
Methods invoke, ainvoke, batch
retriever = store.as_retriever(
    search_type="similarity",
    search_kwargs={"k": 5},
)

docs = retriever.invoke("What is create_agent?")

def format_docs(docs):
    return "\n\n".join(
        f"Source: {d.metadata.get('source')}\n{d.page_content}"
        for d in docs
    )

Best: đánh giá recall riêng, dùng metadata filters và cite source.

Mistake: nhồi quá nhiều docs làm context loãng hoặc không đo retrieval quality.

DL

Document Loaders

Load data from many sources
Khái niệm Load PDF, web, CSV, folder, database thành documents.
Khi dùng Ingest tài liệu trước khi split/embed.
Constructor Provider/package specific loader.
Methods load, lazy_load, load_and_split
from langchain_community.document_loaders import (
    PyPDFLoader,
    WebBaseLoader,
    DirectoryLoader,
)

pdf_docs = PyPDFLoader("report.pdf").load()
web_docs = WebBaseLoader(
    "https://example.com"

# Loaders return list[Document]

Mistake: ingest HTML thô hoặc không xử lý duplicate documents.

TS

Text Splitters

Chunk documents for retrieval
Khái niệm Chia tài liệu dài thành chunks phù hợp embedding/context.
Khi dùng Corpus dài, PDFs, docs, markdown, code.
Constructor RecursiveCharacterTextSplitter(...)
Methods split_text, split_documents, create_documents
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=800,
    chunk_overlap=120,
    add_start_index=True,
)

chunks = splitter.split_documents(pdf_docs)

Best: tune chunk size bằng query thật, giữ overlap vừa đủ.

Mistake: chunk quá nhỏ mất context, quá lớn giảm recall.

DO

Document Object

Core data structure in LangChain
Khái niệm Container cho text và metadata trong RAG.
Khi dùng Loaders, splitters, vector stores, retrievers.
Constructor Document(page_content, metadata, id)
Fields page_content, metadata, id
from langchain_core.documents import Document

doc = Document(
    page_content="LangChain agents call tools.",
    metadata={
        "source": "notes.md",
        "section": "agents",
    },
)

print(doc.page_content)
print(doc.metadata["source"])

Best: metadata cần source, page/section, timestamp để cite và debug.

Mistake: dùng Document như chat message hoặc bỏ metadata nguồn.

AG

Agents & Tools

React, tool-calling, autonomous decision-making
Khái niệm Agent = model + tools + prompt + middleware chạy trên LangGraph.
Khi dùng Model cần tự quyết định gọi tool/API/database nhiều bước.
Constructor create_agent(model, tools, system_prompt, response_format)
Methods invoke, stream, stream_events
from pydantic import BaseModel, Field
from langchain.agents import create_agent
from langchain.tools import tool

class SearchInput(BaseModel):
    query: str = Field(description="Search query")

@tool(args_schema=SearchInput)
def search_docs(query: str) -> str:
    """Search internal docs."""
    return "Relevant snippet about LangChain agents."

agent = create_agent(
    model="openai:gpt-5.5",
    tools=[search_docs],
    system_prompt="Answer briefly. Use tools when needed.",
)

result = agent.invoke({
    "messages": [{"role": "user",
                  "content": "Agent differs from chain?"}]
})
print(result["messages"][-1].content)

Best: tool names snake_case, schema chặt, docstring nói rõ khi nào dùng.

Mistake: quá nhiều tool trùng chức năng hoặc tool side effect không approval.

MS

Messages API

HumanMessage, AIMessage, SystemMessage
Khái niệm Role-based context cho chat models và agents.
Khi dùng Chat history, system policy, tool calling, multimodal blocks.
Classes SystemMessage, HumanMessage, AIMessage
Parameter content, metadata, ids, tool calls.
from langchain.messages import (
    SystemMessage,
    HumanMessage,
)

messages = [
    SystemMessage("You are a Python senior."),
    HumanMessage("Explain Runnable in 2 lines."),
]

response = model.invoke(messages)

Mistake: nhồi toàn bộ history vô hạn vào context window.

RG

Full RAG Pipeline

Complete Retrieval-Augmented Generation with LCEL
Khái niệm Retrieve context ở query time rồi đưa cho model trả lời.
Khi dùng Hỏi đáp trên docs riêng, dữ liệu thay đổi, cần trích nguồn.
Pattern Load -> split -> embed -> store -> retrieve -> answer.
Methods load, split_documents, as_retriever, invoke
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

docs = PyPDFLoader("handbook.pdf").load()
chunks = RecursiveCharacterTextSplitter(
    chunk_size=900,
    chunk_overlap=160,
).split_documents(docs)

store = InMemoryVectorStore(emb)
store.add_documents(chunks)
retriever = store.as_retriever(search_kwargs={"k": 4})

def format_docs(docs):
    return "\n\n".join(
        f"Source: {d.metadata.get('source')}\n{d.page_content}"
        for d in docs
    )

prompt = ChatPromptTemplate.from_template("""
Answer only from context. If missing, say you do not know.
Context:
{context}
Question: {question}
""")

question = "What is the main policy?"
context = format_docs(retriever.invoke(question))
answer = (prompt | model | StrOutputParser()).invoke({
    "context": context,
    "question": question,
})

Best: cite source, evaluate retrieval riêng, thêm reranker khi corpus lớn.

Mistake: để model trả lời ngoài context hoặc không lọc metadata theo quyền user.

LG

LangGraph Basics

Stateful multi-agent workflows
Khái niệm Graph runtime cho stateful agents, checkpoints, control flow.
Khi dùng Workflow nhiều bước, human-in-the-loop, durable execution.
Constructor StateGraph(State)
Methods add_node, add_edge, compile
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    messages: list

def call_model(state: State):
    return {"messages": [
        model.invoke(state["messages"])
    ]}

graph = StateGraph(State)
graph.add_node("model", call_model)
graph.add_edge(START, "model")
graph.add_edge("model", END)
app = graph.compile()

Best: dùng LangChain agent trước; chuyển graph khi cần control flow rõ.

Mistake: tự build graph phức tạp khi create_agent đã đủ.

FS

Few-Shot Prompting

Examples in prompt templates
Khái niệm Đưa ví dụ input/output để model học format và style.
Khi dùng Classification, extraction, rewrite style, formatting khó.
Constructor FewShotPromptTemplate(...)
Parameters examples, example_prompt, suffix
from langchain_core.prompts import (
    FewShotPromptTemplate,
    PromptTemplate,
)

examples = [
    {"input": "happy", "output": "positive"},
    {"input": "late", "output": "negative"},
]

example_prompt = PromptTemplate.from_template(
    "Input: {input}\nLabel: {output}"
)

prompt = FewShotPromptTemplate(
    examples=examples,
    example_prompt=example_prompt,
    suffix="Input: {text}\nLabel:",
    input_variables=["text"],
)

Best: ví dụ ngắn, nhất quán, bao phủ edge cases quan trọng.

Mistake: đưa ví dụ mâu thuẫn hoặc quá dài làm tốn context.

MAP

Quick Decision Map

Pick the right LangChain component
  • Runnable chain
    Prompt -> model -> parser cố định, dễ test.
  • create_agent
    Model cần tự chọn tool hoặc làm nhiều bước.
  • response_format
    App cần JSON/schema đáng tin cậy.
  • Retriever
    Cần lấy docs liên quan tại query time.
  • Memory/checkpointer
    Chat nhiều lượt và cần resume theo thread.
  • Middleware
    Retries, PII filtering, human approval, guardrails.
