LM

LLMs & Chat Models

Core language model wrappers
Concept Standard wrapper for model providers; chat output is usually an AIMessage.
Use when Direct model calls, chains, agents, classification, and extraction.
Constructor ChatOpenAI(model, temperature, max_tokens, timeout)
Methods invoke, stream, batch, with_structured_output
from langchain_openai import ChatOpenAI

model = ChatOpenAI(
    model="gpt-4.1-mini",
    temperature=0.2,
    max_tokens=800,
    timeout=30,
    max_retries=6,
)

# Async invoke
async def run_model():
    msg = await model.ainvoke([
        {"role": "system", "content": "Answer in Vietnamese."},
        {"role": "user", "content": "LangChain la gi?"},
    ])
    
    # Async stream
    async for token in model.astream("Explain LCEL"):
        print(token.content, end="")

Best: Set keys through environment variables, configure timeout/retry, and log usage plus latency.

Mistake: Hard-code API keys or assume every provider supports tool calling the same way.

Prompt Templates

Dynamic prompt construction
Concept Template for system, human, and AI messages with named variables.
Use when Reuse instructions, insert history, and create chains with explicit inputs.
Constructor from_messages, from_template
Methods invoke, format_messages, partial, append
from langchain_core.prompts import (
    ChatPromptTemplate,
    MessagesPlaceholder,
)

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a {role}."),
    MessagesPlaceholder("history"),
    ("human", "{question}"),
])

messages = prompt.invoke({
    "role": "LangChain tutor",
    "history": [],
    "question": "What is Runnable?",

Best: Use clear variable names, specific system prompts, and keep prompts separate from business logic.

Mistake: Use raw f-strings with unsanitized user input or forget required variables.

Output Parsers

Parse and structure LLM output

from langchain_core.output_parsers import (
    StrOutputParser,
    JsonOutputParser,
)

parser = JsonOutputParser()

    "Return JSON only: {topic}\n{format}"

    data = await chain.ainvoke({
        "topic": "retriever",

Best: For important outputs, prefer Pydantic schemas plus structured output.

Mistake: Regex JSON out of natural language or skip parse error handling.

CH

Chains - LCEL

Composable pipeline with pipe operator
Concept Runnable is the shared interface for prompts, models, parsers, retrievers, and custom functions.
Use when Fixed flows such as prompt -> model -> parser, plus map/merge branches.
Constructor RunnableLambda(fn), RunnableParallel(...)
Methods invoke, ainvoke, batch, stream, with_retry
from langchain_core.runnables import RunnableLambda, RunnableParallel
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_template(
    "Summarize {topic} in 3 bullets, Vietnamese."
)

chain = prompt | model | StrOutputParser()

async def run_chains():
    # Async invoke
    result = await chain.ainvoke({"topic": "LangChain Runnable"})
    
    # Async streaming
    async for chunk in chain.astream({"topic": "LangChain Runnable"}):
        print(chunk, end="")

    parallel = RunnableParallel(
        summary=chain,
        upper=RunnableLambda(lambda x: x["topic"].upper()),
    )
    res_parallel = await parallel.ainvoke({"topic": "LCEL"})

Best: Use chains for deterministic tasks that should be easy to test and stream.

Mistake: Use an agent for a simple pipeline or leave input/output types uncontrolled.

ME

Memory

Conversation history management
Concept Short-term memory stores state by conversation thread.
Use when Multi-turn chat, resumable agents, and user context.
Constructor create_agent(..., checkpointer=...)
Parameter thread_id in configurable
# Demo only: InMemorySaver is reset when the process restarts.
from langchain.agents import create_agent
from langgraph.checkpoint.memory import InMemorySaver

agent = create_agent(
    model="openai:gpt-4o",
    tools=[],
    checkpointer=InMemorySaver(),
)

config = {"configurable": {"thread_id": "user-42"}}

async def demo_invoke():
    await agent.ainvoke({"messages": [{"role": "user", "content": "My name is Diem."}]}, config)
    result = await agent.ainvoke({"messages": [{"role": "user", "content": "What is my name?"}]}, config)
# Production: persist short-term memory in Postgres async.
from langchain.agents import create_agent
from langgraph.checkpoint.postgres import PostgresSaver

DB_URI = "postgresql://postgres:postgres@localhost:5432/ai?sslmode=disable"

with PostgresSaver.from_conn_string(DB_URI) as checkpointer:
    checkpointer.setup()  # create checkpoint tables/indexes once

    agent = create_agent(
        model="openai:gpt-4o",
        tools=[lookup_customer, create_ticket],
        checkpointer=checkpointer,
    )

    config = {"configurable": {"thread_id": "tenant-acme:user-42"}}
    
    async def invoke_agent():
        response = await agent.ainvoke(
            {"messages": [{"role": "user", "content": "Open a billing ticket"}]},
            config=config,
        )

Best: Use stable thread IDs, Postgres for production, SQLite for local durable workflows, and summarization/trim middleware for long conversations.

Mistake: Use InMemorySaver in production, skip checkpointer.setup(), or generate a new thread_id for every request.

Embeddings

Text to vector representations
Concept Convert text into vectors for semantic search.
Use when RAG, semantic search, clustering, duplicate detection.
Constructor OpenAIEmbeddings(model=...)
Methods embed_query, embed_documents
from langchain_openai import OpenAIEmbeddings

emb = OpenAIEmbeddings(
    model="text-embedding-3-small"
)

# Async calls
async def run_embeddings():
    query_vec = await emb.aembed_query("What is RAG?")
    doc_vecs = await emb.aembed_documents([
        "Retrieval augments generation.",

Best: Use the same embedding model for indexing and querying, and store the model version.

Mistake: Mix vectors from different embedding models or embed raw HTML/noisy text.

Vector Stores

Store and search embeddings
Concept Store vectors plus metadata and search similar documents.
Use when Knowledge base, semantic search, RAG corpus.

from langchain_core.vectorstores import InMemoryVectorStore

store = InMemoryVectorStore(emb)

async def store_actions():
    await store.aadd_documents(chunks)
    
    # Async search
    docs = await store.asimilarity_search(
        "How do agents use tools?",

Best: Choose a production store with persistence, filtering, and backups.

Mistake: use in-memory store for real production data or missing source metadata.

Document Loaders

Load data from many sources

from langchain_community.document_loaders import (
    PyPDFLoader,
    WebBaseLoader,
)

async def load_all():
    # Sử dụng aload() async để tải không chặn
    pdf_loader = PyPDFLoader("report.pdf")

TS

Text Splitters

Chunk documents for retrieval
Concept Split long documents into chunks suitable for embedding and context windows.
Use when Long corpora, PDFs, docs, Markdown, and code.
Constructor RecursiveCharacterTextSplitter(...)
Methods split_text, split_documents, create_documents
from langchain_text_splitters import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=800,
    chunk_overlap=120,
    add_start_index=True,
)

# Chạy trong background thread
async def chunk_docs(docs):
    return await asyncio.to_thread(splitter.split_documents, docs)

Best: Tune chunk size with real queries and keep overlap only as large as needed.

Mistake: Chunks that are too small lose context; chunks that are too large reduce recall.

AG

Agents & Tools

React, tool-calling, autonomous decision-making
Concept Agent = model + tools + prompt + middleware, executed on LangGraph.
Use when The model must decide which tools, APIs, or databases to call across multiple steps.
Constructor create_agent(model, tools, system_prompt, response_format)
Methods invoke, stream, stream_events
from pydantic import BaseModel, Field
from langchain.agents import create_agent
from langchain_core.tools import tool

class SearchInput(BaseModel):
    query: str = Field(description="Search query")

@tool(args_schema=SearchInput)
async def search_docs(query: str) -> str:
    """Search internal docs."""
    return "Relevant snippet about LangChain agents."

agent = create_agent(
    model="openai:gpt-4o",
    tools=[search_docs],
    system_prompt="Answer briefly. Use tools when needed.",
)

# Async agent execute
async def run_agent():
    result = await agent.ainvoke({
        "messages": [{"role": "user", "content": "Agent differs from chain?"}]
    })
    print(result["messages"][-1].content)

Best: Use snake_case tool names, strict schemas, and docstrings that say when to use each tool.

Mistake: Expose too many overlapping tools or allow side-effect tools without approval.

RG

Full RAG Pipeline

Complete Retrieval-Augmented Generation with LCEL
Concept Retrieve context at query time and pass it to the model for grounded answers.
Use when Q&A over private docs, changing data, and answers that need citations.
Pattern Load -> split -> embed -> store -> retrieve -> answer.
Methods load, split_documents, as_retriever, invoke
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

async def format_docs(docs):
    return "\n\n".join(
        f"Source: {d.metadata.get('source')}\n{d.page_content}"
        for d in docs
    )

prompt = ChatPromptTemplate.from_template("""
Answer only from context. If missing, say you do not know.
Context:
{context}
Question: {question}
""")

async def run_rag(question: str):
    # Retrieve async
    retriever = store.as_retriever(search_kwargs={"k": 4})
    docs = await retriever.ainvoke(question)
    context = await format_docs(docs)
    
    # Chain async invoke
    chain = prompt | model | StrOutputParser()
    answer = await chain.ainvoke({
        "context": context,
        "question": question,
    })
    return answer

Best: Cite sources, evaluate retrieval separately, and add a reranker when the corpus is large.

Mistake: Let the model answer outside context or skip metadata filtering by user permissions.

API

FastAPI Streaming SSE

Expose LangChain streaming chains qua FastAPI StreamingResponse
from fastapi import FastAPI
from fastapi.responses import StreamingResponse
import json

app = FastAPI()

@app.post("/stream")
async def chat_stream(prompt_str: str):
    async def event_generator():
        # Sử dụng astream_events v2 để xuất tokens/events bất đồng bộ
        async for event in chain.astream_events({"topic": prompt_str}, version="v2"):
            if event["event"] == "on_chain_stream":
                chunk = event["data"]["chunk"]
                yield f"data: {json.dumps({'chunk': chunk})}\n\n"
                
    return StreamingResponse(event_generator(), media_type="text/event-stream")

MID

Agent Middleware & Runtime Context

Inject tenant/user context and route model calls centrally
from dataclasses import dataclass
from langchain.agents import create_agent
from langchain.agents.middleware import ModelRequest, ModelResponse, wrap_model_call
from langchain_openai import ChatOpenAI

basic_model = ChatOpenAI(model="gpt-4.1-mini")
premium_model = ChatOpenAI(model="gpt-4.1")

@dataclass
class RequestContext:
    tenant_id: str
    user_tier: str

@wrap_model_call
def route_model(request: ModelRequest, handler) -> ModelResponse:
    ctx = request.runtime.context
    request.model = premium_model if ctx.user_tier == "premium" else basic_model
    return handler(request)

agent = create_agent(
    model=basic_model,
    tools=tools,
    context_schema=RequestContext,
    middleware=[route_model],
)

result = await agent.ainvoke(
    {"messages": [{"role": "user", "content": "Analyze this incident."}]},
    context=RequestContext(tenant_id="acme", user_tier="premium"),
)

API

Advanced API Surface

High-value classes, functions, and methods to know beyond the basics
from langchain_core.runnables import (
    RunnableLambda,
    RunnableParallel,
    RunnablePassthrough,
)
from langchain_core.output_parsers import StrOutputParser

def normalize_question(data: dict) -> str:
    return data["question"].strip()

chain = (
    RunnablePassthrough.assign(
        normalized_question=RunnableLambda(normalize_question),
        context=lambda x: retriever.invoke(x["question"]),
    )
    | prompt
    | model.with_retry(stop_after_attempt=3)
    | StrOutputParser()
).with_config(
    tags=["support", "rag"],
    metadata={"prompt_version": "qa_v4"},
)

parallel = RunnableParallel(
    answer=chain,
    retrieved_docs=lambda x: retriever.invoke(x["question"]),
)
  • Runnable
    invoke, ainvoke, batch, abatch, stream, astream, astream_events, with_retry, with_config.
  • RunnableLambda
    Wrap a Python function as a chain step; use for transforms, validators, and custom routing glue.
  • RunnableParallel
    Run branches in parallel and merge named outputs for multi-input prompts or ensemble retrieval.
  • RunnablePassthrough.assign
    Keep the original input and add derived fields such as retrieved context or computed metadata.
  • ChatPromptTemplate
    from_messages, MessagesPlaceholder, partial, format_messages, prompt composition.
  • BaseChatModel
    bind_tools, with_structured_output, stream, usage metadata, provider-specific model kwargs.
  • Document
    page_content, metadata, id; keep source, page, tenant, and version metadata.
  • VectorStore
    add_documents, delete, similarity_search, similarity_search_with_score, as_retriever.
  • Retriever
    invoke, batch, search types such as similarity, MMR, similarity score threshold, and provider filters.
  • create_agent
    Build tool-calling agents with model, tools, system_prompt, response_format, middleware, and checkpointer.
  • @tool
    Expose Python functions with type hints, docstrings, argument schemas, and controlled side effects.
  • LangSmith
    Trace runs, compare prompts, build datasets, run evaluations, and inspect agent/tool failures.

Advanced Patterns

Structured output, middleware, routing, retries, streaming
from pydantic import BaseModel, Field
from langchain.agents import create_agent
from langchain_core.tools import tool

class TicketTriage(BaseModel):
    priority: str = Field(description="low, medium, high, urgent")
    route_to: str = Field(description="team queue")
    rationale: str

@tool
def lookup_customer(customer_id: str) -> str:
    """Use for customer plan, SLA, and account status lookup."""
    return "enterprise plan, 24/7 SLA"

agent = create_agent(
    model="openai:gpt-4.1",
    tools=[lookup_customer],
    system_prompt=(
        "Triage support tickets. Use tools before assigning SLA. "
        "Return structured output only."
    ),
    response_format=TicketTriage,
)

async def triage_ticket():
    result = await agent.ainvoke({
        "messages": [{"role": "user", "content": "Customer c_123 has outage"}]
    })
    return result

Use agents for: tool selection, multi-step decisions, missing information, and external actions.

Keep chains for: deterministic transforms, extraction, fixed RAG, formatting, and simple classification.

RAG

Advanced RAG

Query rewrite, MMR, rerank, compression, citations
from langchain_core.runnables import RunnableLambda, RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

def format_docs(docs):
    return "\n\n".join(
        f"[{doc.metadata.get('source')}#{doc.metadata.get('page')}]\n"
        f"{doc.page_content}"
        for doc in docs
    )

retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={
        "k": 6,
        "fetch_k": 40,
        "lambda_mult": 0.35,
        "filter": {"tenant_id": tenant_id, "doc_status": "approved"},
    },
)

async def retrieve_context(x: dict) -> str:
    docs = await retriever.ainvoke(x["question"])
    return format_docs(docs)

async def run_advanced_rag():
    rag_chain = (
        RunnablePassthrough.assign(
            context=RunnableLambda(retrieve_context)
        )
        | rag_prompt
        | model
        | StrOutputParser()
    )

    answer = await rag_chain.ainvoke({
        "question": "What is the refund SLA?",
        "tenant_id": "acme",
    })
    return answer
  • MultiQueryRetriever
    Generate query variants to increase recall when users ask vague questions.
  • ContextualCompressionRetriever
    Compress retrieved documents before generation to reduce noise and token cost.
  • EnsembleRetriever
    Blend dense vector retrieval with BM25/keyword retrieval for exact terms plus semantics.
  • search_type="mmr"
    Maximal marginal relevance improves diversity in retrieved chunks.
  • metadata filters
    Filter by tenant, permission, product, date, source type, or document version before answering.
  • citation contract
    Return source ids/pages from Document.metadata; never cite generated text.
PRO

Real-Life LangChain Architecture

How production apps usually compose LangChain pieces
# Typical production shape
request
  -> input validation / auth / tenant context
  -> Runnable chain OR create_agent
  -> retriever / tools / structured output
  -> guardrails / human approval for risky actions
  -> LangSmith trace + eval metadata
  -> response with citations, schema, usage, latency

# Keep these versioned:
prompt_version = "support_triage_v4"
retriever_version = "kb_2026_06_30"
model_policy = {"primary": "openai:gpt-4.1", "fallback": "openai:gpt-4.1-mini"}
  • Stable inputs
    Validate user input, tenant id, permissions, locale, and expected output schema before model calls.
  • Composable core
    Use LCEL for deterministic flows; use agents only when the model must choose tools or iterate.
  • Explicit context
    Pass runtime metadata through config/tags, not hidden globals; preserve traceability.
  • Source contracts
    For RAG, every answer should carry source ids, chunk ids, retrieval score, and index version.
  • Side effects
    Tools that write, send, delete, purchase, or mutate state need confirmation or human approval.
  • Fallbacks
    Handle model timeout, tool failure, empty retrieval, parse failure, and rate-limit separately.
KEY

Keywords

Core LangChain terms and what they mean in real apps
  • Runnable
    Common interface for prompts, models, retrievers, tools, parsers, and composed chains.
  • LCEL
    LangChain Expression Language using |, RunnableParallel, and assign to compose flows.
  • create_agent
    High-level LangChain API for tool-calling agents built on LangGraph runtime.
  • Middleware
    Agent extension point for model routing, tool call limits, context engineering, and guardrails.
  • response_format
    Typed structured output contract for agent final responses.
  • Retriever
    Runnable component that returns relevant documents for RAG.
  • with_retry
    Runnable wrapper for transient failures such as rate limits or network errors.
  • LangSmith
    Tracing, evaluation, debugging, and production observability layer for LangChain apps.
